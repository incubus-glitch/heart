# Heart Disease Classification

Проект посвящён задаче бинарной классификации наличия сердечно-сосудистого заболевания по медицинским показателям пациентов. В рамках работы были выполнены анализ данных, предобработка признаков, визуализация структуры данных и сравнение нескольких моделей машинного обучения.

---

## Описание проекта

В качестве исходных данных использовался набор `heart.csv`, содержащий медицинские показатели пациентов и целевую переменную `target`, которая отражает наличие или отсутствие заболевания сердца.

Основная цель проекта — построить модель, способную по набору признаков предсказывать наличие сердечно-сосудистого заболевания, а также сравнить качество нескольких алгоритмов классификации.

Целевая переменная:

* `target` — наличие заболевания сердца (`1`) или его отсутствие (`0`).

## Что было сделано

В рамках проекта были выполнены следующие этапы:

* загружен и изучен исходный датасет;
* проверены размерность данных, типы столбцов и описательная статистика;
* изучены признаки и их смысловое значение;
* построена визуализация методом главных компонент (`PCA`) только для анализа структуры данных;
* обучены и сравнены несколько моделей классификации;
* подобраны гиперпараметры с помощью `GridSearchCV` и `StratifiedKFold`;
* обнаружены и удалены дубликаты, чтобы исключить завышение качества моделей;
* повторно обучены модели после очистки данных;
* проведено сравнение качества моделей по метрикам `accuracy_score` и `f1_score`.

---

## Используемые методы

### PCA

Использовался только для визуализации структуры данных в двумерном пространстве. Обучение моделей проводилось на исходных признаках.

### StandardScaler

Применялся для масштабирования признаков в модели `LogisticRegression`, так как эта модель чувствительна к масштабу входных данных.

### GridSearchCV

Использовался для подбора гиперпараметров моделей и поиска оптимальной конфигурации.

### StratifiedKFold

Применялся для кросс-валидации с сохранением пропорций классов в каждом фолде.

### Метрики качества

Для оценки моделей использовались:

* **Accuracy** — доля правильных предсказаний;
* **F1-score** — более информативная метрика для бинарной классификации, учитывающая баланс между precision и recall.

В проекте были протестированы следующие модели:

## Результаты

После удаления дубликатов и повторного обучения моделей были получены следующие результаты:

| Модель             | Accuracy | F1-score |

| LogisticRegression | 0.7869   | 0.7860   |
| DecisionTree       | 0.8033   | 0.8028   |
| RandomForest       | 0.7705   | 0.7689   |
| XGBoost            | 0.7705   | 0.7674   |

## Выводы

В ходе работы было показано, что датасет содержит достаточно информации для решения задачи классификации, однако наличие большого числа дубликатов сильно искажало оценку качества моделей. После очистки данных результаты стали более реалистичными.

Наилучший результат в данном эксперименте показала модель `DecisionTreeClassifier`, немного опередив `LogisticRegression`. Это говорит о том, что для данного набора данных важную роль играют нелинейные зависимости между признаками.

При этом логистическая регрессия также показала близкое качество и может рассматриваться как сильный базовый алгоритм для данной задачи.
